# 오늘뭐먹지 — 크롬 수동 테스트 케이스

> **대상 URL**: `http://localhost:3000/ko` (로컬) 또는 배포 URL
> **테스트 환경**: Chrome 최신 버전
> **기록 방법**: 각 케이스 옆 Pass ✅ / Fail ❌ + Fail 원인 메모

---

## 테스트 방법

1. `pnpm dev` 실행 후 Chrome에서 `http://localhost:3000/ko` 접속
2. 각 케이스를 순서대로 테스트
3. **Fail 시** → 원인란에 증상 기록 (예: "버튼 클릭 시 아무 반응 없음", "콘솔 에러 발생")
4. Chrome DevTools(F12) 콘솔 에러도 함께 확인

## 자주 발생하는 Fail 원인 체크리스트

| 증상 | 원인 가능성 | 확인 방법 |
|------|-----------|---------|
| 컴포넌트 미노출 | import 누락, 조건부 렌더링 오류 | F12 → Elements 탭에서 DOM 존재 여부 |
| 클릭 무반응 | onClick 핸들러 오류, disabled 상태 | F12 → Console 에러 확인 |
| 토스트 미표시 | ToastProvider 컨텍스트 누락 | 상위 Provider 래핑 여부 확인 |
| 외부 링크 미오픈 | `target="_blank"` 누락, 팝업 차단 | 브라우저 팝업 차단 해제 |
| 캘린더 미업데이트 | localStorage 파싱 오류 | Application → Local Storage 값 확인 |
| AI 응답 없음 | GEMINI_API_KEY 환경변수 누락 | .env.local 파일 확인 |
| Hydration 에러 | 서버/클라이언트 시간 불일치 | `useMemo()`로 클라이언트 전용 확인 |
